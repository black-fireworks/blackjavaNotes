# java 面向对象
简介
```
面向对象简单来说就是将一系列动作进行封装,多重调用.
换句话来说,例如我们想要队一个人进行描述,面向过程就是一步步去实现人这个定义的所有事情.
而面向对象我们可以对其进行抽象封装将其变为一个对象,将他的各个行为变成方法.
我们也可以对人这实例进行子类实现,也就是我们面向对象的继承,而我们对他的内部方法实现也可以理解为内部封装,定义该对象对外部访问的权限.
```

## 面向对象的三大思想与特性
三大思想
```
OOA(Object Oriented Analysis)
    面向对象分析

OOD(Object Oriented Design)
    面向对象设计

OOP(Object Oriented Programming)
    面向对象程序
```
三大特性
```
封装性
    所有的内容对外部不可见

继承性
    将其他的工程继承下来继续发展

多态性
    方法的重载本身就是一个多态性的体现
```

## 类与对象
简介
```
java 中类类似于我们在制造物品时使用的图纸.(这句话也是有问题的,因为我们类仍然有类属性与类方法,而我们现实中图纸并没有这个概念)
而对象就类似于我们使用图纸造出来的实际产物.
```

在简介中我们说过对象是类的产品,也就是说我们得先有个类那么类如何定义?
```java
/**
 * @author 黑色的小火苗
 * 权限修饰符 class 类名称{
 *     权限修饰符 数据类型 成员属性名称; // 这种写法就是我们常见的对象属性
 *     权限修饰符 static 数据类型 类属性名称;  // 这就是我们常见的类属性
 *
 *     权限修饰符 返回值类型 方法名(参数类型 参数名称){
 *          // 这就是我们常用对象方法的语法格式
 *     }
 *
 *     权限修饰符 static 返回值类型 方法名(参数类型 参数名称){
 *         // 这就是我们常见的类方法的语法
 *     }
 *     
 *     // 总结来说就是我们想要定义对象属性或方法时不适用 static 进行修饰即可
 *     // 反之我们想要定义类属性和类方法时我们则需要使用 static 进行修饰即可
 * }
 * 我们来写一个类.
 */
public class Person {
    /**
     * 这就是我们的对象属性
     */
    public String name;
    /**
     * 这就是我们的类属性
     */
    public static int age;

    private static final  int MAX_AGE = 100;


    /**
     * 这就我们常用的对现象方法
     * @param name 姓名
     */
    public void setName(String name) {
        this.name = name;
    }

    /**
     * 这就是我们常用的静态方法
     * @param age 年龄
     */
    public static void setAge(int age) {
        if (age > MAX_AGE | age < 0) {
            System.out.println("年龄设置错误,请重新设置");
            return;
        }
        Person.age = age;
    }

    
    @Override
    public String toString() {
        return "Person{" +
                "name='" + name + '\'' +
                '}';
    }

    /**
     * 我们现在有了 Person 类,可是我们该如何调用类方法和类属性呢?
     * 我们先来认识类方法和类属性如何调用
     *      类名称.类属性; // 这样就调用了该类的属性
     *      类名称.类方法; // 这样就调用了该类的方法
     * 我们认识了类方法和类属性的调用,我们再来认识下对象属性和对象方法怎么调用.
     *      // 我们需要先实例化一个对象方法如下
     *      类名 对象名 = new 类名();
     *      // 这样我们有了对象名称后就可以调用对象的属性和方法了
     *      对象名.对象属性; // 这样就调用了对象的属性;
     *      对象名.对象方法; // 这样就调用了对象的方法;
     * 我们在 main 方法实际演示
     */
    public static void main(String[] args) {
        // 我们先来调用类的属性和方法
        int age = Person.age;
        // 我们可以看到输出结果为0,因为 int 类型默认值为0
        // 当我们给age设置其他默认值的情况下访问该属性就是我们设置好的默认值
        System.out.println(age);
        // 我们调用类方法 setAge 就为我们类的 age 属性设置了值
        Person.setAge(20);
        // 我们再次访问查看是否成功, 输出后我们可以看到输出结果为 20
        System.out.println(Person.age);

        // 我们再来查看如何调用对象的属性和对象方法.
        Person person = new Person();
        // 我们先来调用对象方法为属性设置数值
        person.setName("张三");
        // 我们输出这个对象来进行查看结果 -->  Person{name='张三'} 我们可以看到属性值设置成功了.
        System.out.println(person);
        // 那我们能不能直接查看属性的值呢? --> 张三 我们可以看到输出结果正常.
        // 但是我们属性使用的时 public 修饰的属性,可我们如果使用的 private 修饰的话就无法直接访问属性值了.
        System.out.println(person.name);
    }
}
```
### 构造器
概述
```
构造器的作用
    用于对象的初始化

执行时间
    在初始化对象时自动调用.

特点
    所有java 类 至少会存在一个构造器方法,如果不写默认会存在一个空参构造器.
    但是如果我们自定义了一个构造器则不会自动创建一个空参构造器.

构造器语法
    // 注意 构造器名称需要与类名称一致
    权限修饰符 构造器名称(){
        // 构造方法
    }
```

### 方法的重载
```
方法名相同,参数类型或参数长度不同,可以理解为方法的重载.
方法的重载与返回值无关.
```
### 构造方法的重载
```
一个类中可以存在多个构造器,我们可以根据使用需求来自定义构造器.
构造器中参数长度不同就是构造器的重载.
```
### 匿名对象
```
没有对象名称的对象就是匿名对象.
匿名对象只能使用一次,用完以后就只能等待 GC 进行回收.
当我们只需要调用哪个一次某个对象时即可使用匿名对象的形式来创建对象.
```

## 抽象类
简介
```
抽象类就是将多个类中同样内容的东西进行提取,抽象为一个抽象类.
抽象类不能被实例化,并且抽象类被继承时必须实现全部的抽象方法否则该子类必须也是抽象类.
抽象类必须使用 public| procted 修饰符修饰.
因为使用 private 会无法让子类继承那样就失去了抽象类的意义. 
当我们不写修饰符的情况下默认为 public.
```
抽象类与抽象方法的语法
```
// 抽象类
abstract class 类名{
    // 抽象方法，只声明而未实现
    public abstract void 方法名(); 
}

```

## 接口
概念
```
当一个类中所有方法都是抽象方法且全部属性都是全局常量那么可以将其定义为接口.
当我们想要实例化一个接口时必须依赖于子类的实现.
```
接口的语法格式
```
interface 接口名称{
    全局常量;
    抽象方法;
}
// 因为接口中只有方法没有实际实现,所以可以多继承.
```
当我们想要实现一个接口时需要使用 `implements` 关键字,实现一个接口的示例如下
```
// 一个类可以实现多个接口,但是只能继承一个对象.'
class 子类名 implements 接口名1,接口名2{
}
```

接口与抽象类的区别
```
1. 抽象类是被子类继承,而接口是需要被子类进行实现.
2. 接口中只能定义声明抽象方法,而抽象类中可以定义抽象方法也可以定义非抽象方法.
3. 接口里定义的变量只能为公共的静态常量,抽象类中的变量是普通变量
4. 抽象类使用的是继承所以只能单继承,但是接口使用的时接口实现可以实现多个接口.
5. 抽象类中可以包含 static 方法,但是接口不允许出现.(因为静态方法不可以被子类方法实现,所以接口中不可以声明静态方法.)
6. 接口中不能有构造方法,但是抽象类可以.
```

## 多态
概念
```
多态就是对象的多种表现形式.

多态的体现
    我们在子类继承父类这就是多态的一种体现形式.
    方法的重载和重写也是多态的一种体现形式(相同方法名的多种形态);
    重载是一个类中方法的多态体现性
    重写是字符类中方法的多态体现性
```
当我们想要判断某个对象是否是某个类的实例我们可以使用 `instanceof` 关键字,语法如下
```
实例化对象 instanceof 类 //此操作返回boolean类型的数据
```

对象类型的转换
```
向上转型:将子类实例变为父类实例
    父类 父类对象 = 子类实例;

向下转型:将父类实例变为子类实例
    子类 子类对象 = （子类）父类实例;
```
